页面框架

使用 .shell 控制最大宽度；顶部 .hero 放产品名、 主导航、全局搜索和少量全局偏好；下面每个主功能用 .panel。 这种结构适合个人工具、知识库、任务流、收藏夹、学习类 app。

列表页模式

推荐顺序是：先加载数据，再做状态过滤/收藏过滤，再做搜索排序，再分页。 CSV 导出应该导出过滤后的全集，而不是当前页，避免用户误以为导出了全部结果。 

const filteredItems = items
  .filter(byScope)
  .filter(bySearch)
  .sort(bySearchScoreOrNewest);

const totalPages = Math.max(1, Math.ceil(filteredItems.length / pageSize));
const visibleItems = filteredItems.slice((page - 1) * pageSize, page * pageSize);

分页组件要点

显示当前范围、总数、当前页、总页数；提供 First/Prev/Next/Last； 支持每页数量切换；搜索、过滤、视图模式变化时回到第 1 页；数据减少时把页码 clamp 到最大页。

视觉语言

这个项目使用暖色到冷色的柔和背景、半透明玻璃 panel、圆角 pill controls、 serif 大标题和紧凑列表。核心原则是：主界面轻、卡片密度高、工具按钮小而明确。

可复用组件清单

Hero, Nav pills, Global search, Panel header, Inventory count, Filter chips, Pagination bar, Compact list row, AI snapshot box, Inline row actions, Empty state, Error/success banner, Bulk import preview。
